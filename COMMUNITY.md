Welcome to the Council Hub community! We're building the open-source coordination layer for multi-LLM collaboration.

Report bugs and request features directly on GitHub:
- **[Bug Report](https://github.com/iksnerd/council-hub/issues/new?labels=bug)** — Something isn't working as expected

## Getting Help

### I'm new to Council Hub
1. Read the [README](README.md) for a quick overview
2. Try the [Quick Start](README.md#quick-start) to run it locally
3. Check the [examples/](examples/) directory for templates and sample requests
4. Check existing [GitHub Discussions](https://github.com/iksnerd/council-hub/discussions) for similar questions

### I found a bug
1. Search [existing issues](https://github.com/iksnerd/council-hub/issues) to see if it's been reported
2. If not found, [open a bug report](https://github.com/iksnerd/council-hub/issues/new?labels=bug) with:
   - Steps to reproduce
   - Expected behavior
   - Actual behavior
   - Your environment (OS, Docker version, etc.)

### I have a feature idea
1. Check if it's already been suggested in [GitHub Discussions](https://github.com/iksnerd/council-hub/discussions?discussions_q=is%3Aopen+category%3AIdeas)
2. If not, [start a discussion](https://github.com/iksnerd/council-hub/discussions/new?category=ideas) to gather feedback
3. Once the idea matures, a maintainer can help turn it into a formal [feature request issue](https://github.com/iksnerd/council-hub/issues/new?labels=enhancement)

### I want to contribute code
1. Check the [Contributing Guide](CONTRIBUTING.md) for development setup
2. Look for [good first issues](https://github.com/iksnerd/council-hub/labels/good%20first%20issue)
3. Or [pick an issue](https://github.com/iksnerd/council-hub/issues) you'd like to tackle
4. Comment on the issue to let us know you're working on it
5. Submit a PR when ready
